搭建简历网站

报名链接：七天训练营
第二课链接：【新手玩转云计算】为求职加分——创建炫酷的简历网页


1. 开通云服务器


有学生认证的同学可以免费领取半年的ECS产品。ECS全称是弹性计算服务Elastic Compute Service，昨天直播老师敲黑板了，要记住！

云服务器 ECS（Elastic Compute Service）是一种弹性可伸缩的计算服务，助您降低 IT 成本，提升运维效率，使您更专注于核心业务创新。
如果没有半年学生免费ECS实例：领取免费ECS资源，30分钟完成基于ECS的博客搭建


计费方式：

一般来说，包年包月适用于长期运行，按量付费适用于测试


地域：

不同地域物理上不相通，开通后无法转换。选择离访问者越近的，速度越快


实例：

此处免费领取的ECS可以是ecs.t5-c1m2.large，2核、4GiB（之前已经领取，忘了还有什么免费选项了）<如何挑选实例>


镜像：

按照课程需要，选择 镜像市场 > LAMP环境（Ubuntu18.04 Apache PHP7.x）


存储：

默认系统盘是高校云盘 40GB，如有需要，可以选择另外购买SSD数据盘



2. 安装网页运行环境


更换镜像

如需更换镜像，先进入控制台，左侧橙色菜单栏进入云服务器ECS，我的资源卡片里点击进入实例。若有数据资源，先快照和备份。

点击右上角 停止 > 强制停止

左侧灰色菜单栏点击实例进入实例列表，右侧点击 更多 > 磁盘和镜像 > 更换操作系统

购买镜像时，选择用户密码模式。


设置安全组

进入实例后，左侧点击 本实例安全组 > 配置规则 > 手动添加 一条入方向访问规则。
入门学习时可以先打开所有端口和授权对象，但不要在上线产品中这么做


配置成功

访问 http://<公网IP>/，检查你的网页是否正常



3. 制作个人简历网页


根据提供的响应式HTML简历模板下载HTML和资源文件

因应个人资料在HTML中修改文本和替换图片。
截一部分我的简历，修改了部分样式（技能条太短了不好看，百分比瞎整的，我还是小白！！！）

4. 将简历上传到服务器​


FileZilla

我们需要通过SFTP协议把文件传输到云服务器上，推荐开源跨平台软件FileZilla。

在主机栏填写sftp://<公网IP>，输入实例的用户名和密码，端口号22 > 快速连接


上传文件

右侧远程站点的目录中，找到/data/wwwroot/default路径，新建一个文件夹preset，把所有现有文件拖拽进preset，并把自己的简历项目文件拖拽至当前目录空白处

即可从 http://<公网IP>/ 访问简历~

基础 Bash命令 | Bash与Python脚本编程（英文）

发表于 2020/7/30 更新于 2020/8/6

Command Line Shell


A. Files and Directories

  1. new directory: mkdir {mynewdir}

  2. go to directory: cd {mynewdir}

  3. show present working directory: pwd

  4. delete empty directory: rmdir {mynewdir}

  5. current directory: .

  6. parent directory: ..

  7. create file: touch {new.txt}

  8. copy file: cp {../my.txt} {.}, cp {my.txt} {my2.txt}

  9. move/rename file: mv {my.txt} {empty.txt}

  10. delete file: rm *, rm *.txt, rm {empty.txt}

  11. change permissions: chmod {+x} {test.py}

  12. change owner: chown {user} {filename} chown {user1} {test.py}

  13. change group: chgrp {group} {filename} chgrp {user1} {test.py}

  14. list entires in directory: ls, ls {mynewdir}

    Add arguments (can join multiple, e.g. -la ):

    • list all (inc. hidden) files -a
    • list directory entries only -d
    • human-readable format -h
    • print index number -i
    • long listing format -l
    • sort in reverse order -r
    • print size -s
    • sort by size -S
    • sort by modification time -t
    • list entries by lines -x
    • sort alphabetically by extension -X

    {permission} {no. of inodes} {owner} {group} {size} {date of mod} {filename}

    -rwxrwxrwx 1 user1 user1 107 Jul 29 22:01 test.py


B. File Operations

  1. show content in std output: cat {new.txt}

  2. write file: nano/vim etc. {filename} nano {new.txt}

  3. count no. of characters, words, lines: wc {new.txt}

  4. text type: file {test.py}

    test.py: ASCII text, with no line terminators

  5. print first 10 lines: head {new.txt}

  6. print last 10 lines: tail {new.txt}

  7. translate: tr {' '} {'\n'}

  8. sort: sort (alphabetically), sort {-nr} (numerically reversed)

  9. terminal paging program (scroll through contents): less (quit with q)

  10. Extract parts each line (by separator after -d) and print the field (of no. after -f): cut -d{' '}, -f{1-} {new.txt} (field counts from 1) (1-: from 1 onwards)


C. Additional Commands

  1. show manual of a command: man {echo} (quit with q)
  2. print to std output: echo {Hello}, echo {"Hello"}
  3. globally search for regular expression and print out: grep {true}
  4. current date: date
  5. list of logged in users: who
  6. computer running duration: uptime
  7. unused memory: free

D. Redirect I/O Streams

  1. redirect std output to new file or overwrite: >

    • {./hello.py} > {hello.txt}, {python hello.py} > {hello.txt}

  2. redirect std output to new file or append: >>

    • {./hello.py} > {hello.txt}

//hello.txt

Hello world to STDOUT

  1. redirect std input from file: <

    • {./hello.py} < {hello.txt}

      Get from STDIN: Write to STDOUT: Hello prepared input in hello.txt
      Traceback (most recent call last):
      File “in.py”, line 3, in
      raise ValueError(“Generating error to STDERR…”)
      ValueError: Generate STDERR…

  2. redirect std input and output

    • {./hello.py} < {hello.txt} > {out.txt}

      //terminal, out.txt

      Get from STDIN: Write to STDOUT: Hello prepared input in hello.txt

    • {./hello.py} < {hello.txt} 2> {err.txt} (0 STDIN, 1 STDOUT, 2 STDERR)

      Get from STDIN: Write to STDOUT: Hello prepared input in hello.txt

      //err.txt

      Traceback (most recent call last):
      File “in.py”, line 3, in 

      raise ValueError("Generate STDERR...")

      ValueError: Generate STDERR…

  3. echo to file

    • echo {"Something to write"} > {write.txt}

E. Pipes and Pipelines

  1. Piping within bash (from stdout to stdin)

    • cat spider.txt | tr ' ' '\n' | sort | uniq -c | sort -nr | head

  2. Piping with external programs

    • cat passage.txt | ./capitalize.py

      Here is my passage,
      It contains small letters only.


F. Signals

  • ping www.example.com

  1. SIGINT: Ctrl+C (prints summary of process and results, then interrupt cleanly)

    ^C

    — www.example.com ping statistics —

    4 packets transmitted, 4 received, 0% packet loss, time 3003ms

    rtt min/avg/max/mdev = 162.498/166.949/169.256/2.665 ms

  2. SIGSTOP: Ctrl+Z (stop without terminating)

    ^Z
    [1]+ Stopped ping www.example.com

  3. SIGTERM: kill (runs a program to terminate thread in another terminal)

    • kill {179} (PID number)

      Terminated


G. Processes

  1. list currently running processes: ps

    • in current terminal for current user ps

    • for all users ps ax

      PID TTY STAT TIME COMMAND

      179 tty3 S 0:00 ping www.example.com

      180 tty4 Ss 0:00 /init

      181 tty4 S 0:00 -bash

    • show environment (variables) ps e

  2. continue process after SIGSTOP: fg

  3. make stopped job to background: bg

  4. list currently running or stopped jobs: jobs

  5. show processes using the most CPU time: top (quit with q)
